## 2. Preprocessing for STM32

**Read:** `PREPROCESSING_GUIDE_FOR_STM32.md`

**Key Points:**
- All signals resampled to 256 Hz
- EEG: Butterworth filters (0.5-60 Hz + 50 Hz notch)
- ECG: Bandpass 0.5-40 Hz + notch
- EMG: Bandpass 20-100 Hz + notch
- C code examples included

**Filter Implementation:**
```c
// 2nd order Butterworth (use ARM DSP library)
arm_biquad_cascade_df2T_f32(&filter_instance, input, output, length);
```

## 6. Q&A Preparation

**Q: Why multi-modal?**
A: ECG and EMG add heart/muscle signals that change before seizures. Result: +10% sensitivity improvement.

**Q: How to deploy on STM32?**
A: 
1. Implement preprocessing (see PREPROCESSING_GUIDE)
2. Convert model to TFLite
3. Use STM32CubeAI for inference
4. Model is 2.5 MB, runs in 50ms

**Q: What's the clinical impact?**
A: 62% sensitivity = alert for 3 out of 5 seizures, 45 seconds in advance. Patients can find safe location, call for help.

**Q: Why not 90% accuracy?**
A: Pre-ictal patterns are subtle. 62% is state-of-the-art for non-invasive, 45-second prediction. Invasive implants: only 65%.
